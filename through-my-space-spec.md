# Through My Space - 仕様書

## アプリ概要

視野狭窄・色覚異常・白内障などの視覚症状を、
**自分の空間を撮影した空間写真を使って体験する**visionOSアプリ。

「他人が用意した風景」ではなく「自分が毎日見ている空間」が変容することで、
視覚障害の当事者への共感と理解を生む。

---

## コンセプト

「あなたが毎日見ているこの空間を、別の目で見てください。」

医療的な正確性よりも「体験のきっかけ」を重視する。
完全な再現を謳わず、「近似的な体験」として謙虚に提示する。

---

## なぜ空間写真なのか

**パススルー（現実の映像リアルタイム加工）**
→ Enterprise APIが必要 → App Store配布不可

**空間写真（撮影済みの画像に加工）**
→ 通常のAPIで可能 → 自由にシェーダーをかけられる

撮影済みの空間写真はただのデータなので、
Metalシェーダーで視覚効果を自由に適用できる。

## 競合との差別化

| 既存アプリ | 弱点 | このアプリ |
|---|---|---|
| Thru My Eyes（visionOS対応） | ストック素材のみ | 自分の空間で体験 |
| iPhoneカメラ型シミュレーター | 小画面・没入感なし | Vision Proで全周没入 |
| VR型（Meta Quest等） | Appleエコシステム外 | visionOSネイティブ |

**「自分の部屋を空間写真で撮影してその目で見る」はどこにも存在しない。**

## 対応する視覚症状

### フェーズ1（MVP）

**1. 視野狭窄（緑内障）**
- 画面周辺部を黒で塗りつぶす
- 残存視野の半径をスライダーで調整
- 「視野の15%を失った状態」など段階を設ける

**2. 色覚異常**
- 赤緑色盲（第1色盲・第2色盲）
- 青黄色盲（第3色盲）
- 全色盲（モノクロ）

### フェーズ2（全症状対応・仕上げ）

**3. 中心暗点（加齢黄斑変性）**
- 画面中央部にぼかし・暗点を発生させる
- 視線を向けるところが見えなくなる逆説的体験

**4. 白内障**
- 全体に白濁フィルター
- 光のハレーション（光源周辺が滲む）

**5. 飛蚊症**
- 半透明の影・糸くずが視野をふわふわ漂う
- 視線の動きに追従するが少し遅れる

**6. 網膜色素変性症**
- 視野狭窄＋夜間視力低下の組み合わせ
- 暗い場所での見え方を再現

### Step 1：空間写真の選択

**ライブラリモード**
PhotosPickerでフォトライブラリから既存の空間写真を選択。

---

### Step 2：体験（症状選択を含む）

空間写真を選択すると即座にFull Immersion Spaceに移行。
空間写真は全天球ではなく約60〜80度の画角しかないため、
球体全体ではなく**前方ドーム状メッシュ**に投影する。

```
投影方法：ドーム状メッシュ（前方のみ）
  → 球体に貼ると背後が空白になり没入感が削がれるため
  → ドームにすることで「見える範囲」に自然に収まる
背景：ドーム外は暗いグラデーションで馴染ませる
テクスチャ：空間写真（左右の目それぞれに対応）
シェーダー：選択した症状をリアルタイムで切り替え
```

**体験時の注意（起動時に表示）**
```
この体験は前方の空間写真を使っています。
軽く頭を動かす程度でお楽しみください。
大きく動くと視差のずれが生じます。
```

**空間に入った状態で症状を選択・切り替える。**
フローティングパネルが視野の端に常駐し、
視線を向けるとフェードインする。

```
[フローティングパネル]

👁  視野狭窄
🎯  中心暗点       ← 視線を向けると展開
🎨  色覚異常
☁️  白内障
✨  飛蚊症

──────────────
強度：━━━━━○── 60%
```

```
球体内径：ユーザーを中心に半径3m
テクスチャ：撮影した空間写真（左右の目それぞれに対応）
シェーダー：選択した症状をリアルタイムで切り替え
```

**操作**
- 視線でフローティングパネルを操作 → 症状を選択・切り替え
- デジタルクラウンを回す → 症状の強度を調整（0%〜100%）
- 症状なしを選ぶ → シェーダーなしの正常視野に戻る
- ピンチ長押し → 写真選択に戻る

**症状の切り替えはシームレスに行う。**
選択した瞬間からシェーダーがクロスフェードで切り替わる。
「正常 → 症状あり」の変化がリアルタイムでわかることが体験の核心。

**症状の重ねがけ（フェーズ2で実装）**
「視野狭窄＋白内障」のように複数症状を同時に適用できるモードを設ける。
高齢者の視覚体験（複数症状の併発）をリアルに再現できる。

**アイトラッキングの活用**
中心暗点・飛蚊症は視線位置（LookAt）を毎フレーム取得してシェーダーに渡す。
「見ようとすればするほど逃げていく」という当事者のもどかしさを再現する。
これはVision Proのアイトラッキングがあって初めて成立するこのデバイスならではの体験。

## 技術的実装の詳細

### 空間写真の取り扱い

空間写真は左目用・右目用2枚の画像が1つのファイル（HEIC/QuickTime）にパッケージされている。
これを分離して左右それぞれの目に異なるテクスチャを見せる必要がある。

```swift
// PHAssetから空間写真の左目用・右目用画像を取り出す
// visionOS 2.0の ViewpointComponent を使って
// 左目には左用テクスチャを貼った球体
// 右目には右用テクスチャを貼った球体
// をそれぞれ出し分けることで立体感を維持する
```

この部分を間違えると立体感が崩れるため、実装の最初に確認・動作検証すること。

### Metalシェーダーの構成

**基本方針：ShaderGraph（Reality Composer Pro）をメインに使い、複雑なロジックのみMetalのCustomNodeで補完する。**
ShaderGraphはリアルタイムプレビューができるため開発効率が高い。

**視野狭窄シェーダー**
```metal
// 画面中心からの距離に応じて周辺を黒にマスキング
// smoothstepで境界を柔らかくする
float dist = length(uv - float2(0.5, 0.5));
float mask = smoothstep(radius, radius + edge, dist);
color = mix(color, float4(0, 0, 0, 1), mask);
```

**色覚異常シェーダー（Brettel 1997 行列変換）**
```metal
// 医学的根拠のある変換行列を使用
// 赤緑色盲（第2色盲・緑錐体なし）
float3x3 deuteranopia = float3x3(
    0.367, 0.280, -0.012,
    0.861, 0.673,  0.043,
   -0.228, 0.047,  0.969
);
color.rgb = deuteranopia * color.rgb;
```

**中心暗点シェーダー（アイトラッキング連動）**
```metal
// 視線位置（LookAt）を毎フレーム取得してcenterパラメータとして渡す
// 「見ようとすればするほど逃げていく」体験を実現する
// ⚠ 視線は微振動するためスムージング処理が必須
//   例：前フレームの視線位置と加重平均を取る（lerp）
// ⚠ 毎フレームgaze取得はパフォーマンス検証が必要
float dist = length(uv - smoothed_gaze_center); // スムージング済み視線位置
float blur_amount = smoothstep(scotoma_size, 0.0, dist) * max_blur;
```

**白内障シェーダー（Bloom効果）**
```metal
// 単純な白濁ではなくBloom効果でハレーションを表現
// 1. 輝度の高い部分を抽出
// 2. ガウシアンブラーをかける
// 3. 元画像に加算合成
// これにより「光の眩しさ」がリアルに伝わる
```

**飛蚊症シェーダー（アイトラッキング連動）**
```metal
// 半透明の影を視線に追従させるが少し遅れて動かす
// 視線位置からわずかにオフセットさせることで
// 「追いかけても逃げる」感覚を再現する
```

### Full Immersion Spaceでの実装

```swift
// 大きな球体を生成して内側から見る
// 法線反転の方法：faceCulling = .none を設定するか
// スケールを [-1, 1, 1] で反転する
let sphere = MeshResource.generateSphere(radius: 3.0)
// CustomMaterial または ShaderGraphMaterial で
// faceCulling = .none を設定して内側を表示する
```

**空間写真の画角について（重要）**

空間写真は全天球（360度）ではなく、約60〜80度の画角を持つ立体写真。
球体に貼り付けると写真の外側（背後など）が空白になる。

対策：
- 背景を完全に黒にする（「視界の外は見えない」という演出として使う）
- 空間写真の端をぼかして自然に黒へ馴染ませる境界処理を入れる

- 症状説明に当事者の声（一人称の短いコメント）を入れる
- 「これは再現ではない」を強くかつ自然に明示する
- 教育用途であることをApp Store説明文・アプリ内で明確化
